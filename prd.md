# 제품 요구사항 정의서 (PRD - Product Requirements Document)

## 1. 프로젝트 개요
- **프로젝트명:** 학습용 웹앱 통합 포털 (Edu-Portal)
- **목적:** 다양한 학습용 웹앱을 한곳에 모아 사용자가 쉽게 탐색하고 직관적으로 사용할 수 있도록 지원하는 웹 포털 구축
- **핵심 가치:**
  - **접근성(Accessibility):** 학번 기반의 간단한 인증을 통한 빠른 진입
  - **윤리성(Ethics):** 생성형 AI 및 디지털 도구 활용에 대한 사전 윤리 교육 및 서약 강제
  - **개인화(Personalization):** 사용자별 학습 통계 및 즐겨찾기 제공

## 2. 타겟 사용자 및 주요 유즈케이스
- **주요 사용자:** 선정고등학교 재학생
- **주요 시나리오:**
  1. 학생이 포털에 접속하여 자신의 학번, 이름, 비밀번호를 입력한다.
  2. 학습 시작 전 반드시 지켜야 할 'AI 핵심 윤리 가이드라인' 6개 조항을 확인하고 각각 동의 체크박스를 선택한다.
  3. 온보딩이 완료되면 메인 대시보드로 진입하여 필요한 학습용 웹앱(예: 영어 단어 암기 앱, 수학 문제 풀이 툴, 타이머 등)을 탐색하고 실행한다.

## 3. 기능적 요구사항 (Functional Requirements)

### 3.1. 인증 및 윤리 온보딩 (Authentication & Onboarding)
- **[FR-01] 사용자 정보 입력:** 사용자는 학번, 이름, 비밀번호를 필수 입력해야 한다.
- **[FR-02] 윤리 가이드라인 표출:** 화면에 다음 6가지 AI 핵심 윤리 강령이 시각적으로 명확하게 표출되어야 한다.
  1. **명확한 활용 목적:** 주도성과 합목적성에 기반한 AI 활용 설정
  2. **주도적 학습 설계:** 질문 전 나의 생각과 아는 내용을 먼저 정리하기
  3. **비판적 검증 습관:** 오류 및 편향성 확인을 위한 교차 검증 생활화
  4. **사고의 확장:** 단답 요구를 넘어 깊이 있는 심화 질문 이어가기
  5. **안전과 관계 유지:** 개인정보 보호 수칙 준수 및 AI와의 정서적 거리 유지
  6. **투명성과 윤리:** AI 활용 부분과 내 생각을 명확히 구분하여 출처 표기
- **[FR-03] 필수 서약 기능:** 각 강령에 대한 개별 체크박스가 존재해야 하며, 6개 항목이 모두 체크되어야만 대시보드 진입 버튼('대시보드로 입장하기')이 활성화된다.

### 3.2. 메인 대시보드 (Main Dashboard)
- **[FR-04] 통합 헤더:** 로고, 통합 검색창, 카테고리 퀵 메뉴, 사용자 프로필 정보가 상단에 상시 노출된다.
- **[FR-05] 개인화 영역:** 사용자의 최근 사용 앱 리스트, 즐겨찾기(북마크) 목록, 더미 학습 통계(방문 횟수, 사용 시간 등)를 시각화하여 보여준다.
- **[FR-06] 앱 검색 및 필터링:** 카테고리별(수학, 어학, 과학, 생산성 등) 필터 버튼 및 키워드 검색을 통해 웹앱 리스트를 필터링할 수 있다.
- **[FR-07] 카드형 레이아웃:** 웹앱들은 카드 UI 형태로 배열되며, 카드에는 앱 아이콘, 앱 이름, 한 줄 요약, 평점/리뷰 수, 핵심 태그, '바로가기' 버튼이 포함된다.

### 3.3. 상세 정보 및 앱 실행 (App Detail & Execution)
- **[FR-08] 앱 상세 모달:** 카드 클릭 시 페이지 이동 없이 팝업(모달) 형태로 앱의 상세 스크린샷, 상세 설명, 유저 리뷰를 제공한다.
- **[FR-09] 웹앱 실행:** '바로가기' 또는 모달 내 '앱 실행하기' 버튼 클릭 시 해당 웹앱이 새 창(New Tab)으로 즉시 실행된다.

### 3.4. 즐겨찾기 및 상태 처리 (Bookmark & State Handling)
- **[FR-10] 즐겨찾기 토글:** 앱 카드 우측 상단의 북마크 아이콘 클릭으로 즐겨찾기 추가/삭제가 가능하며, 상태는 LocalStorage에 즉시 저장되어 페이지 새로고침 후에도 유지된다.
- **[FR-11] 빈 상태(Empty State) UI:** 검색 결과가 없거나 즐겨찾기 목록이 비어있을 때, 안내 메시지 및 아이콘을 포함한 Empty State UI를 표시한다.

## 4. 데이터 및 시스템 제약사항 (Data & System Constraints)
- **[CO-01] DB 미지원 상태 구현 (중요):** 현재 개발 단계에서는 백엔드 데이터베이스(DB) 연결을 지원하지 않는다.
- **[CO-02] 프론트엔드 상태 관리:** 사용자가 입력한 로그인 정보, 윤리 강령 체크 여부, 즐겨찾기 추가/삭제 등 모든 입력 기능은 브라우저의 컴포넌트 상태(State) 또는 `LocalStorage`를 활용하여 전역적으로 유지 및 모킹(Mocking)한다.
- **[CO-03] 데이터 구조 인터페이스화:** 추후 실제 DB(REST API 등) 연결 전환이 용이하도록 데이터 모델 및 CRUD 헬퍼 함수를 독립된 파일/레이어로 분리하여 작성한다.

## 5. 비기능적 요구사항 (Non-Functional Requirements)
- **[NFR-01] 가독성 및 UI 톤앤매너:** 대시보드는 눈의 피로를 줄이는 화이트/라이트 그레이 배경을 기본으로 하며, 신뢰감을 주는 딥 블루(#1e3a8a)를 메인 테마 색상으로 사용한다.
- **[NFR-02] 가치 강화 유저 인터페이스:** 윤리 강령 서약 영역은 줄글 형태가 아닌, 인포그래픽 카드 스타일로 가독성을 극대화하여 학생들이 대충 넘기지 않도록 유도한다.
- **[NFR-03] 반응형 레이아웃:** 모바일(360px~), 태블릿(768px~), 데스크탑(1280px~) 브레이크포인트에 맞는 반응형 UI를 제공하여 다양한 디바이스 환경에서 정상적으로 사용 가능해야 한다.
- **[NFR-04] 웹 접근성:** 키보드 탐색 지원, 이미지 대체 텍스트(alt) 제공, ARIA 레이블 적용 등 WCAG 2.1 AA 수준의 웹 접근성 기준을 준수한다.

## 6. 향후 개선 사항 (Future Improvements, DB 연동 후 반영 예정)
- **[FUT-01] 비밀번호 검증 정책:** 최소 자릿수, 특수문자 포함 등 실제 비밀번호 유효성 검사 규칙을 서버 사이드에서 적용한다.
- **[FUT-02] 세션 만료 정책:** 일정 시간 비활동 시 자동 로그아웃 처리 및 세션 토큰 만료 정책을 서버와 연동하여 정의한다.
- **[FUT-03] 로그아웃 기능:** 헤더 프로필 영역에서 로그아웃 수행 시 서버 세션 및 LocalStorage를 초기화하고 온보딩 화면으로 복귀한다.
